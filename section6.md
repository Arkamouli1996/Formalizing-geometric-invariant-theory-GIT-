# Section 6: `ReynoldsRewrite_S6`

File: [GIT/Week2.lean](GIT/Week2.lean#L311-L440)

## Purpose

This section is **Step 6** of the finite-generation proof for the invariant
subalgebra `R^G` (a.k.a. Hilbert's finiteness theorem for reductive group
actions). It takes an ideal-membership fact living **upstairs in the ambient
ring `R`** and transports it **downstairs into the invariant subalgebra `A`**
by applying the **Reynolds operator** `ρ : R →ₗ[k] A`.

Concretely, it closes the gap between:

- **Upstairs (R):** "the ideal generated by a finite set `s ⊆ R` contains
  (the image of) every `f ∈ A`" — this is the output of Steps 4–5.
- **Downstairs (A):** "the ideal of `A` generated by chosen lifts of `s`
  already contains every such `f`" — this is what Step 7 needs to conclude
  finite generation of `R₊^G`.

The bridge is the Reynolds operator, which is `k`-linear, fixes `A` pointwise
(`ρ ∘ toR = id`), and is `A`-linear via `ρ(toR a · r) = a · ρ r`.

## Setup (section variables)

- `k` — base field.
- `A` — commutative `k`-algebra (eventually `R^G`).
- `R` — commutative `k`-algebra with `Algebra A R` and `IsScalarTower k A R`.
- `toR : A →ₐ[k] R` — inclusion of the invariant subalgebra.
- `ρ : R →ₗ[k] A` — Reynolds operator (`k`-linear).
- `s : Finset R` — finite family of elements of `R` (the generators from Step 4).
- `lift : R → A` — chosen preimages under `toR`.

The hypotheses `hlift`, `hρ_id`, `hρ_mul` capture the three properties above.

## The three theorems

### 1. `reynolds_rewrite` — the rewrite form

**Input.** An equation upstairs in `R`:
```
toR f = ∑ x ∈ s, x * coeff x
```
together with the Reynolds hypotheses.

**Output.** The same equation pushed down to `A`:
```
f = ∑ x ∈ s, lift x * ρ (coeff x)
```

**How the proof works.**
1. Apply `ρ` to both sides of the hypothesis `hf`.
2. The LHS `ρ (toR f)` collapses to `f` by `hρ_id`.
3. `ρ` commutes with `Finset.sum` via `map_sum` (because `ρ` is `k`-linear),
   giving `f = ∑ x ∈ s, ρ (x * coeff x)`.
4. For each summand, rewrite `x = toR (lift x)` and then use
   `hρ_mul (lift x) (coeff x) : ρ (toR (lift x) * coeff x) = lift x * ρ (coeff x)`.

   A subtle point: the proof instantiates `hρ_mul` first and then rewrites
   with `hlift` **inside that hypothesis**, rather than rewriting `x` on the
   goal. Why: doing `rw [← hlift x hx]` on the goal would also hit the `x`
   appearing inside `coeff x`, breaking the proof. Rewriting inside the
   hypothesis avoids that.

### 2. `mem_ideal_span_lift_of_reynolds` — ideal-membership form

**Input.** Same hypotheses as `reynolds_rewrite` for a single `f ∈ A`.

**Output.**
```
f ∈ Ideal.span ((lift '' s) : Set A)
```

**How the proof works.**
1. Apply `reynolds_rewrite` to rewrite `f` as a finite sum
   `∑ x ∈ s, lift x * ρ (coeff x)` in `A`.
2. `Ideal.sum_mem` reduces the goal to showing each summand lies in the ideal.
3. Each summand is `lift x * ρ (coeff x)`, a right-multiple of the generator
   `lift x`; use `Ideal.mul_mem_right`.
4. `lift x` itself lies in the span because `x ∈ s` implies `lift x ∈ lift '' s`
   (`Set.mem_image_of_mem`), and `Ideal.subset_span` finishes.

### 3. `RGplusA_le_span_lift_of_reynolds` — the ideal-containment corollary

**Input.** An ideal `RGplusA : Ideal A` together with a *uniform* presentation
hypothesis:
```
∀ f ∈ RGplusA, ∃ coeff : R → R, toR f = ∑ x ∈ s, x * coeff x
```
plus the Reynolds hypotheses. (The presentation hypothesis is exactly the
output of Steps 4–5 applied to the augmentation ideal of the invariants.)

**Output.**
```
RGplusA ≤ Ideal.span ((lift '' s) : Set A)
```

**How the proof works.**
- Pointwise: take any `f ∈ RGplusA`, extract `coeff` from `present`, and
  feed it to `mem_ideal_span_lift_of_reynolds`.

## Role in the larger proof

This is the final "descent" step before Step 7. After this section:

- Steps 4–5 have produced a finite set `s ⊆ R` generating the upstairs ideal.
- Step 6 (this section) shows the lifts of `s` generate the corresponding
  ideal downstairs in `A = R^G`.
- Step 7 uses this together with a graded Nakayama-style argument to conclude
  that `A` itself is a finitely generated `k`-algebra.
